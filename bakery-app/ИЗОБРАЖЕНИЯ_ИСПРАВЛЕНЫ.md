# ✅ Исправление системы изображений товаров

## Проблема
Изображения товаров не соответствовали названиям - отображались неправильные фотографии для разных товаров.

## Решение
Добавлен единый маппинг локальных изображений `PRODUCT_IMAGES` во все компоненты, где отображаются товары.

## Изменённые файлы

### 1. `app/(tabs)/home.tsx`
- ✅ Исправлен маппинг `PRODUCT_IMAGES` с правильными названиями файлов
- ✅ Все товары используют локальные изображения через ключи (croissant, cinnabon, и т.д.)

### 2. `app/(tabs)/cart.tsx`
- ✅ Добавлен полный маппинг `PRODUCT_IMAGES`
- ✅ Изменено отображение: вместо `{ uri: ... }` используется `PRODUCT_IMAGES[key]`
- ✅ Добавлен fallback на `croissant` если изображение не найдено

### 3. `app/(tabs)/orders.tsx`
- ✅ Расширен маппинг `PRODUCT_IMAGES` со всеми 24 товарами
- ✅ Заказы из БД правильно отображают изображения через ключи

### 4. `app/(tabs)/favorites.tsx`
- ✅ Добавлен полный маппинг `PRODUCT_IMAGES`
- ✅ Избранные товары используют локальные изображения
- ✅ При добавлении в корзину передаётся правильный ключ изображения

## Как это работает

1. **Главная страница**: Товары используют ключи из `PRODUCT_IMAGES` (например, `image: 'croissant'`)
2. **Корзина**: При добавлении товара передаётся ключ, который маппится на локальное изображение
3. **Заказы**: При сохранении в БД сохраняется ключ, при загрузке - маппится на изображение
4. **Избранное**: Товары из БД содержат ключ в поле `image_url`, который маппится на локальное изображение
