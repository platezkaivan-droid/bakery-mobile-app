# КРИТИЧЕСКИЕ ОШИБКИ - РЕШЕНИЕ

## Проблема 1: Отсутствует react-native-keyboard-controller

### Решение:
```bash
cd bakery-app/bakery-mobile-app/bakery-app
npm install react-native-keyboard-controller
npx expo prebuild --clean
```

## Проблема 2: Google Services - неправильный package name

В файле `google-services.json` указан package name `com.bakery.pavlova`, 
но в `app.json` у тебя другой.

### Решение:
1. Открой Firebase Console
2. Добавь новое Android приложение с правильным package name из `app.json`
3. ИЛИ измени package name в `app.json` на `com.bakery.pavlova`
4. Скачай новый `google-services.json`

## Проблема 3: Слишком длинные пути (Windows MAX_PATH)

Путь `C:\Users\Administrator\Documents\bakery-app\bakery-mobile-app\bakery-app\node_modules\...` 
слишком длинный для Windows (>250 символов).

### КРИТИЧЕСКОЕ РЕШЕНИЕ:
Перенеси проект в корень диска:

```bash
# Закрой Android Studio и все терминалы
# Перенеси проект:
move C:\Users\Administrator\Documents\bakery-app C:\bakery-app

# Теперь работай из C:\bakery-app\bakery-mobile-app\bakery-app
```

## Проблема 4: compileSdk версия

### Решение:
Обнови `android/app/build.gradle`:

```gradle
android {
    compileSdk 35  // было 34
    
    defaultConfig {
        targetSdk 35  // было 34
    }
}
```

## ЧТО ДЕЛАТЬ СЕЙЧАС:

### Шаг 1: Перенеси проект
```bash
move C:\Users\Administrator\Documents\bakery-app C:\bakery-app
cd C:\bakery-app\bakery-mobile-app\bakery-app
```

### Шаг 2: Установи недостающий пакет
```bash
npm install react-native-keyboard-controller
```

### Шаг 3: Обнови compileSdk
Открой `android/app/build.gradle` и измени:
- `compileSdk 35`
- `targetSdk 35`

### Шаг 4: Исправь google-services.json
Скачай правильный файл из Firebase Console с нужным package name.

### Шаг 5: Чистая пересборка
```bash
cd android
./gradlew clean
cd ..
npx expo prebuild --clean
cd android
./gradlew assembleRelease
```

## ВАЖНО:
Короткий путь (C:\bakery-app) решит 90% проблем с CMake/ninja!
