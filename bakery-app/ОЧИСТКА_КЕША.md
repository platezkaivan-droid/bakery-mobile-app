# Инструкция по очистке кеша

## Проблема
Если вы видите ошибку `your_project.supabase.co` или другие старые данные, это означает, что браузер использует закешированную версию приложения.

## Решение

### 1. Очистка кеша Metro Bundler (сервера)

```bash
# Остановите сервер (Ctrl+C)

# Удалите папку кеша
# Windows PowerShell:
Remove-Item -Recurse -Force .expo

# Запустите с полной очисткой
npx expo start -c --clear
```

### 2. Очистка кеша браузера

#### Способ 1: Жесткая перезагрузка (рекомендуется)
- **Windows/Linux**: `Ctrl + Shift + R` или `Ctrl + F5`
- **Mac**: `Cmd + Shift + R`

#### Способ 2: Очистка через DevTools
1. Откройте DevTools (F12)
2. Кликните правой кнопкой на кнопку обновления страницы
3. Выберите "Очистить кеш и жесткая перезагрузка" (Empty Cache and Hard Reload)

#### Способ 3: Полная очистка кеша
1. Откройте настройки браузера
2. Найдите "Очистить данные браузера" / "Clear browsing data"
3. Выберите:
   - ✅ Кешированные изображения и файлы
   - ✅ Файлы cookie и данные сайтов
4. Временной диапазон: "За все время" / "All time"
5. Нажмите "Очистить данные"

### 3. Проверка

После очистки:
1. Закройте все вкладки с `localhost:8081`
2. Откройте новую вкладку
3. Перейдите на `http://localhost:8081`
4. Откройте консоль браузера (F12) и проверьте, нет ли ошибок

### 4. Если проблема сохраняется

Попробуйте открыть в режиме инкогнито:
- **Windows/Linux**: `Ctrl + Shift + N`
- **Mac**: `Cmd + Shift + N`

Если в инкогнито работает, значит проблема точно в кеше обычного режима.

## Проверка правильности настроек Supabase

Убедитесь, что в файле `src/lib/supabase.ts` указаны ваши реальные данные:

```typescript
const SUPABASE_URL = 'https://qkyhwdmhkoizxjazwnti.supabase.co';
const SUPABASE_ANON_KEY = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...';
```

✅ Ваши данные уже правильно настроены!
