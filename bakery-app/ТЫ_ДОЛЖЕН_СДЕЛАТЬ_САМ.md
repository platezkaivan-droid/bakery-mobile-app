# ЧТО ТЫ ДОЛЖЕН СДЕЛАТЬ САМ

## ⚠️ КРИТИЧЕСКИ ВАЖНО: Перенос проекта

Твой путь слишком длинный для Windows:
```
C:\Users\Administrator\Documents\bakery-app\bakery-mobile-app\bakery-app\
```

Это вызывает 90% ошибок с CMake/ninja!

### СДЕЛАЙ ЭТО ПРЯМО СЕЙЧАС:

1. **Закрой Android Studio и все терминалы**

2. **Перенеси проект в корень диска:**
   ```cmd
   move C:\Users\Administrator\Documents\bakery-app C:\bakery-app
   ```

3. **Открой новый терминал в новой папке:**
   ```cmd
   cd C:\bakery-app\bakery-mobile-app\bakery-app
   ```

4. **Запусти скрипт исправления:**
   ```cmd
   ДЕЛАЙ_СЕЙЧАС_ИСПРАВЛЕНИЕ.bat
   ```

## Проблема с google-services.json

В файле указан package name: `com.bakery.pavlova`

Проверь в `app.json` - какой у тебя package name?

### Если package name другой:
1. Открой Firebase Console
2. Добавь новое Android приложение с правильным package name
3. Скачай новый `google-services.json`
4. Замени файл в `android/app/google-services.json`

### ИЛИ измени package name в app.json на:
```json
{
  "expo": {
    "android": {
      "package": "com.bakery.pavlova"
    }
  }
}
```

## После переноса проекта:

1. Запусти `ДЕЛАЙ_СЕЙЧАС_ИСПРАВЛЕНИЕ.bat`
2. Открой проект в Android Studio из новой папки `C:\bakery-app\bakery-mobile-app\bakery-app\android`
3. Build → Generate Signed Bundle / APK → APK → Release

## Почему это важно?

Windows имеет ограничение MAX_PATH = 260 символов.
Твой путь + node_modules + вложенные папки = >300 символов.
Это вызывает все ошибки "mkdir failed", "path too long", "ninja error".

Короткий путь = все заработает!
